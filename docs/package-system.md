# Package & Chart System

Gorge 使用 `.zip` 格式的包文件来分发游戏代码和资源。包文件分为两类：Runtime Package（运行时库）和 Chart Package（谱面文件）。

## Package 结构

```csharp
class Package {
    List<SourceCodeFile> SourceCodeFiles;  // .g 源码文件
    List<AssetFile> AssetFiles;            // 资源文件（图片、音频等）
}
```

每个包文件是一个 ZIP 归档，内部按路径组织 `.g` 源码和其他资源（PNG、WAV 等）。加载时自动分类：
- `.g` 文件 → SourceCodeFile
- 其他文件 → AssetFile

### 加载方式

```csharp
// 从 ZIP 文件字节加载
Package.LoadZipPackage(byte[] zipFileBinary, bool isChart)

// 从文件路径加载
Package.LoadZipPackage(string zipFilePath, bool isChart)

// 从 Stream 加载
Package.LoadZipPackage(Stream zipFileBinary, bool isChart)

// 从文件夹加载（用于开发调试）
Package.LoadFolderPackage(string folderPath, bool isChart)
```

**`isChart` 参数**：
- `true` — 标记包内容为谱面代码和资源
- `false` — 标记包内容为运行时库代码和资源

`GamePlayer` 在 `PrepareRuntime()` 中通过此标记区分两类包：运行时包提供类型定义（类/接口/枚举），谱面包提供游戏逻辑（Note/Element 等）和媒体资源。

### ZIP 加载细节

`LoadZipPackage` 使用 SharpZipLib 读取 ZIP：
1. 遍历 ZIP 条目
2. `.g` 文件：读取为 UTF-8 字符串（自动处理 BOM）
3. 其他文件：读取为 byte[]
4. 根据 `isChart` 参数标记 `IsChartSourceCode` / `IsChartAsset`

## SourceCodeFile

```csharp
class SourceCodeFile {
    string Path;         // 源文件在包内的相对路径
    string Code;         // 源码内容
    bool IsChartSourceCode; // 是否为谱面代码
}
```

所有 `SourceCodeFile` 的 `Code` 汇总后交给 `Compiler.Compile()` 编译。

## AssetFile

```csharp
class AssetFile {
    string Path;         // 资产在包内的相对路径
    byte[] Data;         // 资产二进制数据
    bool IsChartAsset;   // 是否为谱面资产
}
```

资产通过 `SimulationScore.ExtractAssetsFromPackage()` 提取到 `Score` 中，在 `PrepareScore()` / `ReloadAssets()` 中通过 `Score.AddFileAsset()` 转换为资产注入器和 `Score.LoadAssets()` 加载。

## 谱面系统

### Package 在 GamePlayer 中的生命周期

```
用户配置
├── RuntimePackagePaths  (string[])  → 运行时 .zip 包路径列表
├── ChartPackagePaths    (string[])  → 谱面 .zip 包路径列表
└── Packages             (List<Package>) → 手动添加的 Package 对象

PrepareRuntime()
├── 加载所有 Runtime/Chart 包路径 → Package.LoadZipPackage()
├── 合并手动添加的 _manualPackages
├── 验证：至少 1 个 Runtime 包 + 1 个 Chart 包
├── RuntimeManager.CreateLanguageRuntime(Packages)
│   └── Compiler.Compile(sourceCodeFiles) → ClassImplementationContext
│       └── GorgeLanguageRuntime(native, compiled)
├── RuntimeManager.ExtractSimulationResources()
│   └── Score.ExtractAssetsFromPackage() + PrepareScore()
└── RuntimeManager.CreateSimulationRuntime(onTerminate)
    └── Base.Instance.CreateSimulationRuntime() → GorgeSimulationRuntime
```

### SimulationScore

`SimulationScore` 存储谱面的完整时间线数据：

```csharp
class SimulationScore {
    float BeginChartTime;     // 谱面起始时间
    float TerminateChartTime; // 谱面终止时间
    float BeginSimulateSpeed; // 初始仿真速度
    // + 资产列表 + 谱面 Staff 列表
}
```

### 谱面组织结构

Gorge 的谱面使用层级结构：

```
SimulationScore
├── AudioStaff         ← 音频轨道
│   └── AudioPeriod    ← 按时间段划分的音频片段
├── ElementStaff       ← 元素轨道
│   └── ElementPeriod  ← 按时间段划分的 Note/Element
└── Staff (基类) / IStaff (接口)
```

- **Staff** = 一条独立的轨道（音频/元素/特效等）
- **Period** = 时间段内的数据单元
- 每条 Period 关联一组 Gorge Element 实例

## 运行时形态管理 (RuntimeFormContainer)

`RuntimeFormContainer` 在运行时解析 Gorge 注解元数据：

### 模态 (Form)

通过 `[Form(name, version)]` 注解的静态方法，返回该模态支持的 Element 类型列表：

```csharp
// 在 .g 代码中：
// [Form(name="Standard", version="1.0")]
// static string[] StandardForm() { ... }

// 运行时读取：
Forms["Standard"] = { Name, Version, ElementTypes[] }
```

### Element 修改器

所有继承 `GorgeFramework.Element` 的类，其标注了 `[PeriodModifier]` 的静态方法被收集：

```csharp
ElementModifiers["ClassName"].Modifiers → List<MethodInformation>
```

### Element 生成方式

通过构造方法上的注解确定生成策略：

| 注解 | 含义 |
|------|------|
| `[InitializeGenerate]` | 谱面开始时一次性生成 |
| `[ForwardTimedGenerate]` | 正转时按时间代理计算的时间生成 |
| `[BackwardTimedGenerate]` | 反转时按时间代理计算的时间生成 |

### 即时音效

通过 `[InstantAudio(name)]` 注解的静态方法被收集为即时音效：

```csharp
InstantAudioMethods["effect_name"] = (GorgeClass, MethodInformation)
```

## Native 代码包扫描 (GorgeNative)

`GorgeNative` 类使用反射扫描当前 AppDomain 中所有程序集，查找标记了特定属性的字段：

```csharp
// 扫描 [GorgeNativeClass]
NativeClasses → List<GorgeClass>

// 扫描 [GorgeNativeEnum]
NativeEnums → List<GorgeEnum>

// 扫描 [GorgeNativeInterface]
NativeInterface → List<GorgeInterface>
```

这些 Native 类型和编译后的用户代码类型合并为 `GorgeLanguageRuntime` 的完整类型系统。

Native zip 包 (`res://addons/gorgeplugin/Native.zip`) 的源码位置在 `System/Native/AutoGenerated/` 下，包含：
- `Gorge` 命名空间：Injector, IntArray, FloatArray, BoolArray, StringArray, ObjectArray, IntList, FloatList, BoolList, StringList, ObjectList
- `GorgeFramework` 命名空间：Vector2, Vector3, ColorArgb, Node, Element, Sprite, CurveSprite, NineSliceSprite, Audio, Graph, Math, Random, Logger, 各种 FunctionCurve 类型等

## 计分系统

`GorgeFramework/Stage/` 包含计分相关类：

| 类 | 职责 |
|----|------|
| `ScoringV1` / `IScoring` | 计分算法接口 |
| `ScoringCounter` | 连击/判定计数 |
| `ScoreRecord` | 分数记录（时间戳 + 分数） |
| `ScoreMilepost` | 分数里程碑 |
| `GameResult` | 游戏结算结果 |
| `IScoreDisplayer` | 分数显示接口 |
